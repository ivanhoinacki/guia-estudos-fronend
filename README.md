<h4>Guia do iniciante dev web</h4>
<p>A baixo segue alguns links para melhor enfase no aprendizado <a href="http://tableless.com.br/tornar-dev-front-end/" target="_blank" rel="external">post no Tableless</a> escrito pelo <a href="https://twitter.com/diegoeis" target="_blank" rel="external">Diego Eis</a> (um dos mestres da área) citando o post como referência.</p><h2 id="Como-estudar-para-me-tornar-um-Desenvolvedor-Front-End"><a href="#Como-estudar-para-me-tornar-um-Desenvolvedor-Front-End" class="headerlink" title="Como estudar para me tornar um Desenvolvedor Front End"></a>Como estudar para me tornar um Desenvolvedor Front End</h2><h3 id="Livros"><a href="#Livros" class="headerlink" title="Livros"></a>Livros</h3><ul><li><a href="http://www.maujor.com/index.php" target="_blank" rel="external">Maujor</a></li></ul><p>O cara é simplesmente a maior referência nacional quando se trata de HTML e CSS.</p><p>Se você começou agora e nunca viu uma linha de HTML e CSS recomendo comprar de cara esse livro:</p><p><a href="http://www.maujor.com/livro/livro.html" target="_blank" rel="external">Construindo Sites com CSS e (X)HTML</a>.</p><p>Ele aborda muito bem o básico que você precisa saber sobre HTML e CSS e vai te dar uma base muito boa.</p><p>Boa parte do conteúdo dele está no <a href="http://www.maujor.com/index.php" target="_blank" rel="external">site do Maujor</a>, porém o livro agrupa o conteúdo de uma forma que fica simples de entender.</p><p>Vale a pena dar uma olhada nos outros livros também.</p><ul><li><a href="http://www.oreilly.com/" target="_blank" rel="external">O’Reilly</a></li></ul><p>Em minha humilde opinião, a melhor editora da área. Todos os livros são fantásticos e é impressionante a gama de assuntos que eles abordam. Os livros geralmente são em inglês, porém já existem alguns em português.</p><p>Seguem dois que considero essenciais: <a href="http://www.novateceditora.com.br/livros/guerreiroseo/" target="_blank" rel="external">Guerreiro SEO</a> e <a href="http://www.submarino.com.br/produto/112167569/livro-javascript-o-guia-definitivo" target="_blank" rel="external">Javascript - O Guia Definitivo</a>, porém pouco comoventes.</p><p>Se quiser aprender bem algo, pesquise por “algo” + o’reilly no google e compre o livro.</p><ul><li><a href="http://resrc.io/list/10/list-of-free-programming-books/" target="_blank" rel="external">Lista de livros gratuitos postada na reSRC</a></li></ul><h3 id="Organizacoes"><a href="#Organizacoes" class="headerlink" title="Organizações"></a>Organizações</h3><ul><li><a href="https://developer.mozilla.org/pt-BR/" target="_blank" rel="external">Mozilla Developer Network - MDN</a></li></ul><p>A galera da MDN é extremamente didática e mantém uma documentação abrangente com exemplos de códigos e imagens.</p><ul><li><a href="http://www.w3fools.com/" target="_blank" rel="external">W3Fools</a></li></ul><p>Uma péssima fonte e que ocupa as primeiras posições do google com certa frequência é o W3Schools (não vou desperdiçar meu juice linkando eles).<br>Podem servir de quebra galho as vezes mas não são muito confiáveis e não tem vínculo algum com a consagrada <a href="http://www.w3.org/" target="_blank" rel="external">W3C</a>.</p><ul><li><a href="http://www.w3.org/" target="_blank" rel="external">W3C</a></li></ul><p>Aliás, a W3C obviamente é uma fonte totalmente segura de conhecimento já que ela regula os padrões web.</p><h3 id="Biblia"><a href="#Biblia" class="headerlink" title="Bíblia"></a>Bíblia</h3><ul><li><a href="http://stackoverflow.com/" target="_blank" rel="external">Stack Overflow</a></li></ul><p>Também conhecido como a bíblia do desenvolvedor. Trata-se de um site de Q&amp;A (perguntas e respostas) com uma comunidade muito ativa.</p><p>Se você esbarrou em um erro e não conseguiu sair dali de forma alguma, certamente sua dúvida já foi respondida no Stack Overflow. Caso tenha procurado e nenhum dos resultados do google é deles é um claro indicativo que você está procurando pelo termo errado.</p><p>Obs: Dica de ouro.
